# Деплой — ссылка «зайти и пользоваться», но без доступа к коду

Задеплоенное приложение даёт проверяющим **рабочую ссылку**: они открывают её, загружают
отчёт, смотрят аналитику — но **исходный код не виден и не скачивается** (на сервере крутится
только собранное приложение). Репозиторий при этом можно держать приватным и никого не пускать.

> Важно про модель хостинга: распознавание отчёта занимает 20–40 сек. Поэтому нужен
> **Node-сервер** (Render / Railway), а не «serverless» с коротким лимитом. На **Vercel Hobby**
> лимит функции 10 сек — анализ будет обрываться (подойдёт только платный Vercel Pro с лимитом 60 сек).

## Вариант 1 — Render (рекомендую, бесплатно)
1. Зарегистрируйтесь на [render.com](https://render.com) и подключите GitHub.
2. **New → Blueprint** → выберите репозиторий `testbankrotstvo` (в нём есть `render.yaml`).
3. В настройках сервиса добавьте переменную окружения **`GEMINI_API_KEY`** = ваш ключ.
4. Deploy. Через несколько минут получите ссылку вида `https://credit-analyzer.onrender.com`.
5. Эту ссылку отдаёте проверяющим. Репозиторий им открывать **не нужно**.

Нюансы бесплатного тарифа: сервис «засыпает» после 15 мин простоя (первый заход ~30–60 сек
на пробуждение); загруженные заявки не хранятся вечно (файловая система сбрасывается при
передеплое) — для показа этого достаточно.

## Вариант 2 — Railway (тоже быстро)
1. [railway.app](https://railway.app) → **New Project → Deploy from GitHub repo**.
2. Переменные: `GEMINI_API_KEY`, `SEED_DEMO=true`. Build `npm run build`, Start `npm start`.
3. Получите публичную ссылку в разделе Settings → Networking.

## Вариант 3 — быстрый временный туннель (без деплоя)
Если нужно показать «здесь и сейчас», а приложение уже запущено локально (`npm run dev`):
- установите `cloudflared` и выполните `cloudflared tunnel --url http://localhost:3000`
  — получите временную публичную ссылку на **локальное** приложение (код остаётся на вашем ПК).
- Минусы: ПК должен быть включён и сервер запущен; ссылка временная.

## Безопасность деплоя
- Ключ Gemini задаётся **в панели хостинга** (env var), в код/git не попадает.
- У приложения нет авторизации — любой со ссылкой может им пользоваться (и тратить ваш API-лимит).
  Для ревью это не проблема; после проверки сервис можно **остановить/удалить** или сменить ключ.
