# КредитАнализ — ИИ-система для менеджеров по банкротству физлиц

Веб-система, которая **распознаёт кредитные отчёты** (Кредистори / ОКБ / НБКИ), считает
**сколько клиент переплачивает банкам** и наглядно сравнивает три сценария —
**продолжать платить / банкротство / реструктуризация**. Встроенный **ИИ-ассистент**
погружён в процесс: отвечает менеджеру по конкретной заявке, по процедуре банкротства и по
аналитике воронки. Всё работает **локально на одном ПК**.

> Движок распознавания — **Google Gemini** (`gemini-3.5-flash`). Все денежные расчёты
> считаются детерминированно в коде, поэтому цифры не «выдуманы» моделью.

---

## ✨ Возможности

- **Аналитика / аудит** (`/`) — воронка по статусам, конверсия, сумма проанализированных долгов,
  сколько клиенты уже отдали в проценты, потенциальная экономия, распределение рекомендаций ИИ,
  график поступления заявок, последние заявки.
- **Новая заявка** (`/new`) — перетащите PDF-отчёт, Gemini распознаёт все договоры за ~20–30 сек.
- **Карточка клиента** (`/applications/[id]`):
  - текущий статус, общий долг, кредитный рейтинг;
  - **сколько уже выплачено** и какая доля ушла в проценты, а не в тело долга;
  - **сравнение трёх сценариев** (платёж/мес, срок, итого заплатит, экономия);
  - **таблица решений** — что показать клиенту;
  - **блок compliance** — рисковые сигналы (0–2 платежа, кредиты в течение 4 дней, крупные
    займы с короткой историей, просрочки);
  - таблица распознанных договоров и **аудит** всех действий по заявке.
- **ИИ-ассистент** (`/assistant` и панель в каждой заявке) — знает процесс, видит данные заявки
  и цифры воронки, помогает с формулировками и отработкой возражений.
- **Заявки** (`/applications`) — список с фильтрами по статусу и поиском; статусы меняются в один
  клик, каждое изменение пишется в аудит.
- **Демо-режим** — без ключа система показывает 6 обезличенных демо-заявок, чтобы можно было
  сразу «пощупать» интерфейс.

## 🚀 Запуск за 3 шага

Нужен **Node.js 18.18+** (проверено на Node 26).

```bash
# 1. Установить зависимости
npm install

# 2. Создать .env.local из шаблона и вписать ключ Google Gemini
cp .env.example .env.local
#   затем откройте .env.local и вставьте GEMINI_API_KEY=<ваш ключ из https://aistudio.google.com/apikey>

# 3. Запустить
npm run dev
```

Откройте **http://localhost:3000**. Для продакшн-сборки: `npm run build && npm start`.

> Без ключа приложение запускается в **демо-режиме** (живое распознавание и чат отключены,
> но интерфейс и демо-заявки доступны).

## 🧠 Почему Gemini `gemini-3.5-flash`

Выбор модели проверялся **эмпирически на реальных отчётах** (см. раздел «Как выбиралась модель»):

| Задача | Модель | Почему |
|---|---|---|
| Распознавание PDF-отчётов | **`gemini-3.5-flash`** | Нативно «видит» PDF (таблицы, сложная вёрстка), стабильная GA-модель. На реальном 32-страничном отчёте: **~21 сек, ~$0.03**, все договоры и суммы распознаны верно. |
| Чат-ассистент | `gemini-3.5-flash` | Максимальное «понимание» процесса; при локальной нагрузке стоимость незначительна. |

Обе модели меняются одной строкой в `.env.local`
(`GEMINI_MODEL_EXTRACT`, `GEMINI_MODEL_CHAT`). Как более дешёвые альтернативы для чата подходят
`gemini-3.1-flash-lite` или `gemini-2.5-flash`; как запасной вариант извлечения — `gemini-2.5-pro`.

**Ключевое архитектурное решение:** в модель отправляется **исходный PDF**, а не заранее
извлечённый текст — так сохраняется структура таблиц и не теряются данные из сложной вёрстки.
Модель возвращает **строго типизированный JSON** (`responseSchema`, `temperature: 0`), а всю
математику (сценарии, сроки, экономию) считает код — модель не «фантазирует» с числами.

## 🏗️ Архитектура

```
app/
  page.tsx                 Дашборд / аналитика
  new/page.tsx             Загрузка отчёта
  applications/            Список и карточка заявки
  assistant/page.tsx       Глобальный ИИ-ассистент
  help/page.tsx            Справка и FAQ
  api/
    analyze/route.ts       PDF → Gemini → анализ → сохранение
    chat/route.ts          Стриминговый чат с контекстом
    applications/[id]      Смена статуса / заметки / удаление
lib/
  analyze.ts               Извлечение данных из PDF (Gemini + responseSchema)
  finance.ts               Сценарии, таблица решений, compliance (детерминированно)
  chat.ts                  Системный промпт + стриминг ответа
  knowledge.ts             База знаний процесса (погружение ИИ)
  store.ts                 Локальное хранилище (JSON на файл) + аудит
  analytics.ts             Агрегаты для дашборда
  gemini.ts / types.ts / format.ts / demo.ts
components/                UI: дашборд, анализ, чат, загрузка, список
```

Поток данных: **PDF → `analyze.ts` (Gemini) → `finance.ts` (расчёты) → `store.ts` (JSON) → UI**.
Чат получает контекст из `knowledge.ts` (процесс) + текущей заявки + сводки аналитики.

## 🔒 Данные и приватность

- Заявки хранятся **локально** в `data/applications/*.json` — не покидают ПК и **не попадают в git**
  (см. `.gitignore`).
- Ключ Gemini (`.env.local`) и любые `*.txt` с ключами исключены из git.
- Демо-заявки — **синтетические, с вымышленными ФИО**; реальные отчёты из `данные для ДЗ`
  распознаются только на вашем ПК.

## ⚠️ Ограничения и допущения

- **Расчёты сценариев — оценочные.** Для «продолжать платить» используется амортизация по ПСК;
  для револьверных линий без графика берётся горизонт по умолчанию **36 мес**. Реальные графики
  банков могут отличаться.
- **Стоимость услуг** «Банкротство» (200 000 ₽) и «Реструктуризация» (150 000 ₽), длительность
  банкротства (6 мес) и срок реструктуризации (36 мес) — **параметры по умолчанию**, задаются в
  `.env.local` (`BANKRUPTCY_COST`, `RESTRUCTURING_COST`).
- Система рассчитана на **отчёты в рублях** формата Кредистори / ОКБ (совместимо с НБКИ). На других
  форматах качество распознавания может снижаться.
- **Не является юридической консультацией.** Пороги банкротства и последствия — справочные,
  финальное решение принимает юрист.
- Однопользовательский локальный инструмент: **без авторизации и многопользовательского доступа**.
- Точность извлечения зависит от качества PDF; ИИ может ошибиться на нестандартных отчётах —
  ключевые цифры стоит перепроверять.

## 🧪 Как выбиралась модель (воспроизводимо)

Список доступных моделей и живое извлечение проверялись напрямую по Gemini API вашим ключом:
`gemini-3.5-flash` дал корректный структурированный JSON по реальному отчёту за ~21 сек при
~17K входных токенов. Скрипты проверки — в истории разработки (`тех файлы`).
